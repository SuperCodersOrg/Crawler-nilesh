# Goal

The primary objective for today's development was to integrate all the individual modules of the web crawler into a complete crawling pipeline and stabilize its execution. The focus was on implementing recursive crawling, improving URL processing, filtering unwanted resources, and resolving the runtime issues that prevented the crawler from executing continuously.

---

# Problem

Although each module—including the CDP Downloader, HTML Parser, URL Normalizer, and Frontier—worked correctly in isolation, integrating them exposed multiple logical and runtime issues.

The crawler encountered several problems during execution, including:

- Runtime exceptions caused by invalid URL processing.
- Incorrect duplicate URL detection.
- Relative URL normalization failures.
- Resource filtering edge cases.
- Compilation and linker issues.
- Frequent `std::bad_alloc` exceptions after crawling multiple pages.

Initially, the memory allocation error appeared to be caused by excessive memory usage, but further investigation revealed that sufficient memory was available. This indicated that the actual problem was related to memory corruption inside one of the custom STL data structures.

# What I Tried

## Integrated the Complete Crawling Pipeline

Connected all major crawler components into a single workflow.

Implemented the following execution sequence:

1. Normalize the seed URL.
2. Download rendered HTML using Chrome DevTools Protocol (CDP).
3. Parse hyperlinks from the downloaded HTML.
4. Normalize every extracted URL.
5. Filter invalid domains and unsupported resources.
6. Insert valid URLs into the Frontier.
7. Continue crawling recursively until the configured depth.

# Problems Encountered

### Runtime Exception (`std::out_of_range`)

Invalid URLs caused `substr()` to access positions beyond the string length.

**Resolution**

Added validation before every substring operation and returned immediately for invalid or empty URLs.

---

### Relative URL Handling

Relative URLs were not being converted correctly into absolute URLs.

**Resolution**

Implemented dedicated relative URL conversion before normalization.

---

### Duplicate URL Detection

URLs waiting inside the Frontier were incorrectly marked as already visited.

**Resolution**

Separated queued URLs from crawled URLs to maintain proper duplicate detection.

---

### Resource Filtering

URLs containing query parameters were bypassing extension filtering.

Example:

```
style.css?v=4
```

This behaviour was identified and documented for future refinement.

---

### Header Redefinition

Multiple header inclusions caused compilation errors.

**Resolution**

Added `#pragma once` to all project headers.

---

### Linker Configuration

The project was linked as a Windows GUI application, resulting in a missing `WinMain` error.

**Resolution**

Updated the build configuration to compile the project as a console application.

---

### Heap Corruption (`std::bad_alloc`)

Incorrect pointer deletion inside the LinkedList corrupted heap memory, eventually causing allocation failures.

**Resolution**

Fixed the remove logic by deleting the correct node, updating pointers safely, and validating the implementation through repeated testing.
